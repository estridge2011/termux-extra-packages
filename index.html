Termux extra packages

There are located packages which are not added into the official Termux repositories.

How to enable this repository

  1. You may need to install a package dirmngr before adding a GPG key. In Termux it is recommended to use a wrapper script called pkg for executing APT package manager, so it will be used in examples.
pkg install dirmngr
  1. Add GPG key to the APT’s keyring. Note that this command may sometimes fail (especially if Internet connection is bad).
apt-key adv --keyserver pool.sks-keyservers.net --recv 9D6D488416B493F0
  1. Add these lines to sources.list file. This file is located in ‘${PREFIX}/etc/apt’.
# Extra packages by Xeffyr
deb https://termux.xeffyr.ml extra main
  1. Update local APT’s database of packages. This step is not necessary, but you may do it to verify that everything is okay.
pkg update

Source code for these packages

You can get sources on project’s Github page (xeffyr/termux-extra-packages). Note that tarballs are not stored, only build environment and patches.
